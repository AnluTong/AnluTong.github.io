KD tree

发表于 2016-10-13   |   分类于 点云处理

简介

KD tree（K dimensional树的简称），是一种分割K维数据空间的二叉树。主要应用于多维空间关键数据的搜索（如：范围搜索和最近邻搜索）。

描述

多维空间关键数据搜索大致可分为两类：一类是线性扫描法，即将数据集中的点与查询点逐一进行距离比较，缺点很明显，搜索效率较低；第二类是建立数据索引，然后再进行快速匹配。因为实际数据一般都会呈现出簇状的聚类形态，通过设计有效的索引结构可以大大加快检索的速度。索引树属于第二类，其基本思想就是对搜索空间进行层次划分。根据划分的空间是否有混叠可以分为Clipping和Overlapping两种。前者划分空间没有重叠，其代表就是k-d树；后者划分空间相互有交叠，其代表为R树。

KD tree算法可以分为两大部分，一部分是有关KD tree本身这种数据结构建立的算法，另一部分是在建立的KD tree上如何进行最（K）邻近查找的算法。

KD tree构建

先以一个简单的实例来介绍KD Tree构建算法。假设有6个二维数据点{(2,3),(5,4),(9,6),(4,7),(8,1),(7,2)}，数据点位于二维空间内，如下图。构建算法就是要确定图中分割空间的分割线（多维空间即为分割平面，一般为超平面）。

有了直观的了解之后，接下来我们来描述KD Tree的构建算法。KD tree构建是一个递归过程，步骤如下：

  1. 确定分割数据的维度，选取分割值（超平面），分割值存储到当前节点
  2. 其余数据一分为二，分别进入左右子树中。
  3. 重复步骤1和2，直到数据不可分。

有两个细节问题需要搞清，一是分割维度如何选择，二是分割值的选择。

分割维度选择，每次对子空间的划分时，怎样确定在哪个维度上进行划分？最简单暴力的方式是“轮着来”，可采用当前树深度对总维度数求模。当数据分布较为均匀时，该方法较为简洁有效，而当分布不均匀时，该方法会降低索引效率。为了解决这个问题，我们引入方差的概念。数学上来说，某维度数据方差比较大，那么这些数据在该维度上分散的比较开，我们就更容易在这个维度上将它们划分开。因此合理的策略是每次我们选择维度进行划分时，都选择具有最大方差的维度。

分割值选择，在某个维度上进行划分时，怎样确保在这一维度上的划分得到的两个子集合的数量尽量相等？方法很简单，以该维度值为键值排序，取中间的数（即中位数，median），然后将所有数据的键值与中位数比较，按大小划分可以得到两个子集数据，这样得到的两个子集合数据个数基本相同。

以前文二维数据的例子，对KD Tree构建进行简要分析：

1.确定分割维度。分别计算x，y方向上数据的方差得知x方向上的方差最大，所以分割维度首先取0；

2.确定分割值。根据x维度方向的值2,5,9,4,8,7排序选出中值为7，所以分割节点为(7,2)。该节点的分割超平面就是通过(7,2)并垂直于x轴的直线x = 7；

3.确定左右子树。分割超平面x = 7将整个空间分为两部分，x < 7的部分为左子树数据，包含3个节点{(2,3),(5,4),(4,7)}；另一部分为右子树数据，包含2个节点{(9,6),(8,1)}。

如算法所述，KD tree的构建是一个递归的过程。然后对左右子树的重复上面的过程就可以得到下一级子节点(5,4)和(9,6)，同时将空间和数据集进一步细分。如此反复直到空间中只包含一个数据点，最后生成的KD tree如下图所示。

K 近邻搜索

当K值取1时，K近邻搜索算法就演化成最近邻搜索算法。下面，我们先对最近邻搜索算法进行分析。

首先目标点从根结点开始，按照与各结点的比较结果向下访问KD tree，直至达到叶子结点。

目标点与结点的比较指的是当前划分维度上的目标值与结点值比较，若更小则访问左子树，否则访问右子树。达到叶子结点时，计算目标点与叶子结点上保存的数据之间的距离，记录下最小距离对应的数据点，记为当前“最近邻点”和最小距离。

然后进行回溯操作，该操作是为了找到更近的“最近邻点”。

欲判断当前“最近邻点”的父节点未被访问过的分支是否存在更近的“最近邻点”，只需判断以目标点为圆心，当前“最近邻点”为半径的球面，是否与经过父节点的分割超平面相交。具体来说，就是计算父节点划分维度下，目标节点与父节点差的绝对值。该值若小于当前最小距离，则未访问的分支（包括父节点）可能存在更近的“最近邻点”。

若未访问的分支可能存在更近的“最近邻点”，重复第一步，向下访问，直至达到叶子节点，然后重复第二步，进行查找。

若另一分支不存在更近的“最近邻点”，则回溯到父节点，重复第二步，直到回溯到根结点。

搜索过程示意，下面用动态图展示：

K近邻搜索，我们采用BBF（Best-Bin-First）改进方法进行近似搜索。《从K近邻算法、距离度量谈到KD树、SIFT+BBF算法》这篇博客对改进BBF算法的描述是：
1.在某一层，分割面是第ki维，分割值是kv，那么 abs(q[ki]-kv) 就是没有选择的那个分支的优先级，也就是计算的是那一维上的距离；
2.同时，从优先队列里面取节点只在某次搜索到叶节点后才发生，计算过距离的节点不会出现在队列的，比如1~10这10个节点，你第一次搜索到叶节点的路径是1-5-7，那么1，5，7是不会出现在优先队列的。换句话说，优先队列里面存的都是查询路径上节点对应的相反子节点，比如：搜索左子树，就把对应这一层的右节点存进队列。

我们基于大顶堆设计了优先级队列，K近邻搜索过程总共需要两个优先级队列，一个存储已经访问的路径结点，另一个存储候选结点，两者队列容量均为K。迭代步骤如下：

  1. 根节点入候选队
  2. 候选队出队
  3. 搜索出队结点直到叶节点，搜索过程中将访问路径结点和候选结点（访问结点的兄弟分支）分别入队
  4. 重复步骤2，3直到候选队列为空。
